Temporal dashboard inputs

Panel B, Forecast one year ahead

The frontend loads the first available continuous forecast file from these paths:

temporal_prediction/temporal_predictions_all_years.csv
temporal_prediction/temporal_country_generalisation_all_predictions.csv
temporal_prediction/temporal_matched_predictions_all_years.csv

Equivalent files in the dashboard root are accepted as fallbacks.

The continuous forecast file must provide Country, Species, Family, target_year, model_name, a predicted susceptibility column, and a previous susceptibility column.

Panel C, Flag what may change

The frontend loads:

temporal_prediction/temporal_jump_candidate_rankings.csv

The root level temporal_jump_candidate_rankings.csv is accepted as a fallback.

Required jump fields:

Country
Species
Family
input_year
target_year
model_name
direction
score
score_type
score_is_probability
rank_country
predicted_alert
prospective

Accepted directional models:

frozen_residual_jump_heads
frozen_residual_jump_heads_future
gnn
gnn_future

Only rows with score_type equal to learned_probability_score, score_is_probability equal to true, and score between zero and one are displayed.

The frontend does not derive jump scores from predicted susceptibility and does not recalculate ranks or alerts.
